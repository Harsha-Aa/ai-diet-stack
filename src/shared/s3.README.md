# S3 Client Utility

A comprehensive S3 utility module for the AI Diet Meal Recommendation System. This module provides a singleton S3 client with helper functions for common S3 operations, including pre-signed URL generation for secure uploads and downloads.

## Features

- **Singleton Pattern**: Reuses S3 client across Lambda invocations for optimal performance
- **Pre-signed URLs**: Generate secure, time-limited URLs for client-side uploads/downloads
- **Helper Functions**: Simplified API for common S3 operations
- **TypeScript Support**: Full type safety with TypeScript definitions
- **Connection Pooling**: Automatic connection reuse for better performance
- **Retry Logic**: Built-in retry with exponential backoff

## Usage Examples

### Pre-signed Upload URLs (Mobile App Uploads)

Generate a pre-signed URL that allows mobile apps to upload files directly to S3:

```typescript
// In your Lambda function
import { generatePresignedUploadUrl } from './shared/s3';

export const handler = async (event: APIGatewayProxyEvent) => {
  const userId = event.requestContext.authorizer?.userId;
  const foodId = ulid();
  const contentType = 'image/jpeg';
  
  // Generate pre-signed URL for mobile app
  const uploadUrl = await generatePresignedUploadUrl(
    process.env.FOOD_IMAGES_BUCKET!,
    `users/${userId}/foods/${foodId}.jpg`,
    contentType,
    {
      expiresIn: 300, // 5 minutes
      metadata: {
        userId,
        foodId,
        uploadedAt: new Date().toISOString(),
      },
    }
  );
  
  return {
    statusCode: 200,
    body: JSON.stringify({
      uploadUrl,
      foodId,
      expiresIn: 300,
    }),
  };
};
```

**Mobile app usage:**
```typescript
// Mobile app code (React Native, Flutter, etc.)
const response = await fetch(apiUrl + '/foods/upload-url');
const { uploadUrl, foodId } = await response.json();

// Upload file directly to S3
```

### Pre-signed Download URLs

Generate a pre-signed URL for downloading files:

```typescript
import { generatePresignedDownloadUrl } from './shared/s3';

export const handler = async (event: APIGatewayProxyEvent) => {
  const userId = event.requestContext.authorizer?.userId;
  const foodId = event.pathParameters?.foodId;
  
  // Generate pre-signed URL for download
  const downloadUrl = await generatePresignedDownloadUrl(
    process.env.FOOD_IMAGES_BUCKET!,
    `users/${userId}/foods/${foodId}.jpg`,
    {
      expiresIn: 3600, // 1 hour
      responseContentType: 'image/jpeg',
      responseContentDisposition: `attachment; filename="food-${foodId}.jpg"`,
    }
  );
  
  return {
    statusCode: 200,
    body: JSON.stringify({ downloadUrl }),
  };
};
```

### Direct File Upload (Lambda to S3)

Upload files directly from Lambda (e.g., generated reports):

```typescript
import { uploadFile } from './shared/s3';

export const handler = async (event: any) => {
  const userId = event.userId;
  const reportId = ulid();
  
  // Generate report (PDF, CSV, etc.)
  const reportBuffer = await generateReport(userId);
  
  // Upload to S3
  const etag = await uploadFile(
    process.env.REPORTS_BUCKET!,
    `users/${userId}/reports/${reportId}.pdf`,
    reportBuffer,
    'application/pdf',
    {
      metadata: {
        userId,
        reportId,
        generatedAt: new Date().toISOString(),
      },
      cacheControl: 'max-age=3600',
      serverSideEncryption: 'AES256',
    }
  );
  
  console.log(`Report uploaded with ETag: ${etag}`);
};
```

### Check File Existence

Check if a file exists before processing:

```typescript
import { fileExists } from './shared/s3';

export const handler = async (event: any) => {
  const userId = event.userId;
  const foodId = event.foodId;
  
  const exists = await fileExists(
    process.env.FOOD_IMAGES_BUCKET!,
    `users/${userId}/foods/${foodId}.jpg`

    return {
      statusCode: 404,
      body: JSON.stringify({ error: 'Food image not found' }),
    };
  }
  
  // Process the file...
};
```

### Get File Metadata

Retrieve file metadata without downloading the entire file:

```typescript
import { getFileMetadata } from './shared/s3';

export const handler = async (event: any) => {
  const userId = event.userId;
  const foodId = event.foodId;
  
  const metadata = await getFileMetadata(
    process.env.FOOD_IMAGES_BUCKET!,
    `users/${userId}/foods/${foodId}.jpg`
  );
  
  return {
    statusCode: 200,
    body: JSON.stringify({
      size: metadata.ContentLength,
      contentType: metadata.ContentType,
      lastModified: metadata.LastModified,
      customMetadata: metadata.Metadata,
    }),
  };
};
```

## Configuration

### Pre-signed URL Expiration Times

The module provides default expiration times:

```typescript
import { PRESIGNED_URL_EXPIRATION } from './shared/s3';

PRESIGNED_URL_EXPIRATION.UPLOAD;   // 300 seconds (5 minutes)
PRESIGNED_URL_EXPIRATION.DOWNLOAD; // 3600 seconds (1 hour)
PRESIGNED_URL_EXPIRATION.SHORT;    // 60 seconds (1 minute)
PRESIGNED_URL_EXPIRATION.LONG;     // 86400 seconds (24 hours)
```

You can override these by passing custom `expiresIn` values to the functions.

### Environment Variables

Ensure these environment variables are set in your Lambda functions:

```typescript
process.env.FOOD_IMAGES_BUCKET // S3 bucket for food images
process.env.REPORTS_BUCKET     // S3 bucket for reports
```

These are automatically set by the CDK infrastructure.

## CORS Configuration

The food images bucket is configured with CORS to allow mobile app uploads:

- **Allowed Methods**: GET, PUT, POST
- **Allowed Origins**: `*` (restrict in production)
- **Allowed Headers**: Content-Type, Authorization, X-Amz-* headers
- **Max Age**: 3600 seconds (1 hour)

## Security Best Practices

### 1. Use Pre-signed URLs for Client Uploads

Instead of uploading through Lambda:
```typescript
// ❌ Bad: Upload through Lambda (expensive, slow)
const imageBuffer = await readFileFromRequest(event);
await uploadFile(bucket, key, imageBuffer, contentType);

// ✅ Good: Generate pre-signed URL for direct upload
const uploadUrl = await generatePresignedUploadUrl(bucket, key, contentType);
return { uploadUrl };
```

### 2. Set Appropriate Expiration Times

```typescript
// ✅ Short expiration for uploads (5 minutes)
await generatePresignedUploadUrl(bucket, key, contentType, { expiresIn: 300 });

// ✅ Longer expiration for downloads (1 hour)
await generatePresignedDownloadUrl(bucket, key, { expiresIn: 3600 });
```

### 3. Include User Context in Object Keys

```typescript
// ✅ Good: User-scoped keys
const key = `users/${userId}/foods/${foodId}.jpg`;

// ❌ Bad: Global keys (security risk)
const key = `foods/${foodId}.jpg`;
```

### 4. Add Metadata for Tracking

```typescript
await uploadFile(bucket, key, buffer, contentType, {
  metadata: {
    userId,
    uploadedAt: new Date().toISOString(),
    source: 'mobile-app',
  },
});
```

### 5. Use Server-Side Encryption

```typescript
await uploadFile(bucket, key, buffer, contentType, {
  serverSideEncryption: 'AES256', // or 'aws:kms'
  sseKmsKeyId: process.env.KMS_KEY_ID, // for KMS encryption
});
```

## Performance Tips

### 1. Singleton Client

The module automatically uses a singleton S3 client that's reused across Lambda invocations. No additional configuration needed.

### 2. Connection Pooling

Connection pooling is handled automatically by the AWS SDK. The client is configured with:
- Connection timeout: 3 seconds
- Request timeout: 30 seconds
- Max retry attempts: 3
- Retry mode: adaptive

### 3. Prefer Pre-signed URLs

For large files, always use pre-signed URLs to avoid Lambda payload limits and reduce costs:

```typescript
// ✅ Good: Pre-signed URL (no Lambda payload limit)
const uploadUrl = await generatePresignedUploadUrl(bucket, key, contentType);

// ❌ Bad: Direct upload through Lambda (6MB payload limit)
await uploadFile(bucket, key, largeBuffer, contentType);
```

## Error Handling

All functions throw errors that should be caught and handled:

```typescript
try {
  const buffer = await downloadFile(bucket, key);
  // Process buffer...
} catch (error) {
  if (error.name === 'NoSuchKey') {
    return { statusCode: 404, body: 'File not found' };
  }
  console.error('S3 error:', error);
  return { statusCode: 500, body: 'Internal server error' };
}
```
